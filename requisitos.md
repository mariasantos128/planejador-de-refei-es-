Lista de Requisitos Funcionais
1. Gestão de Usuários e Acessos
RF01 - Cadastro de Administrador: O sistema deve permitir que um usuário se cadastre como Administrador da conta familiar. 👑

RF02 - Convite de Membros: O Administrador deve poder enviar convites (via e-mail ou link) para outros membros da família.

RF03 - Controle de Permissões: O sistema deve restringir a edição de refeições e receitas apenas ao perfil Administrador. Membros convidados possuem acesso apenas para visualização e interação com a lista de compras.

RF04 - Perfil de Restrições: O sistema deve permitir registrar restrições alimentares no perfil (ex: Glúten, Lactose). ⚠️

2. Gestão de Receitas
RF05 - Banco de Receitas Padrão: O sistema deve ser entregue com uma base inicial de receitas comuns para facilitar o uso imediato. 🍝

RF06 - Cadastro de Receitas Personalizadas: O Administrador deve poder criar novas receitas informando nome, ingredientes, quantidades, modo de preparo e categoria.

RF07 - Edição de Receitas: O Administrador deve ter permissão para editar qualquer receita (padrão ou personalizada).

RF08 - Categorização e Busca: O sistema deve permitir filtrar receitas por categorias (ex: Vegetariana, Rápida, Favoritas).

RF09 - Alerta de Restrições: Ao visualizar uma receita que contenha itens marcados nas restrições do perfil, o sistema deve exibir um aviso visual de atenção.

3. Planejamento Alimentar
RF10 - Cronograma Semanal: O sistema deve apresentar uma interface de calendário semanal dividida por tipos de refeição (Café, Almoço, Jantar, Lanches). 📅

RF11 - Atribuição de Refeições: O Administrador deve poder selecionar receitas e atribuí-las a dias e horários específicos da semana.

RF12 - Edição Rápida (Clicar e Trocar): O sistema deve permitir que o Administrador altere uma refeição planejada clicando sobre ela e selecionando uma substituta.

4. Lista de Compras
RF13 - Geração Automática: O sistema deve gerar uma lista de compras consolidando todos os ingredientes necessários para o plano semanal ativo.

RF14 - Organização por Setores: A lista de compras deve ser agrupada automaticamente por categorias de mercado (Hortifruti, Açougue, Laticínios, etc.). 🍎🥩

RF15 - Interatividade da Lista: O usuário deve poder marcar itens como "comprados" ou excluir manualmente itens que já possui em casa antes de ir ao mercado. ✅

Priorização Sugerida
Alta: RF01, RF02, RF05, RF06, RF10, RF11, RF13 (O "coração" do sistema).

Média: RF03, RF07, RF08, RF12, RF14, RF15 (Melhorias de usabilidade e gestão).

Baixa: RF04, RF09 (Personalização e segurança alimentar).
